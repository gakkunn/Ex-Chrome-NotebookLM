# NotebookLM Shortcut Effective Extension

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](./LICENSE)

## Overview

A Chrome extension that boosts NotebookLM input and navigation with shortcuts, prevents accidental sends, and adjusts the layout. You can toggle each feature and customize key bindings from the popup.

This is an open-source extension implemented based on Chrome Extension Manifest v3.

---

## Features

- Safe Send: Enter inserts a newline; Cmd/Ctrl + Enter sends (toggleable)
- Vim-like Scroll: j / k / Shift+j / Shift+k / Cmd/Ctrl + j/k to scroll or jump to top/bottom
- Wide Screen & Omnibar Toggle: Shift+Space switches focus/visibility to free up a wider workspace
- Workflow Shortcuts: Keyboard shortcuts to open the Sources tab and click "Add source," clear chat history (wide view), show/hide sidebars, and switch tabs in small view
- Shortcut Help & Customization: Edit shortcuts in the popup; press Cmd/Ctrl + / to show the in-page help modal

### 2. Install dependencies & Build

```bash
npm install

```

### 3. Install to Chrome (Developer Mode)

1. Open Chrome  
2. Go to `chrome://extensions/`  
3. Toggle **"Developer mode"** on in the top right corner  
4. Click **"Load unpacked"**  
5. Select the `dist/` (build output) or project root if you are running `npm run dev`

## Usage

1. Pin the extension icon and open NotebookLM (`https://notebooklm.google.com/`).
2. Example actions:
   - Shift + Space toggles omnibar focus/visibility (when the wide screen toggle is enabled)
   - Cmd/Ctrl + Enter sends; Enter inserts a newline only (Safe Send)
   - j / k / Shift+j / Shift+k / Cmd/Ctrl + j/k for scrolling (Vim-like Scroll)
   - Cmd/Ctrl + U opens the Add Sources dialog; Cmd/Ctrl + Shift + Delete clears chat history (wide view)
   - Cmd/Ctrl + , / . toggles sidebars; on small screens it switches tabs
   - Cmd/Ctrl + / opens the shortcut list modal
3. Click the toolbar icon to open the popup, toggle features, or edit shortcuts. Settings are saved to Chrome storage and applied immediately to the NotebookLM page. Use **Reset** to restore the defaults.

## Project Structure

```text
notebooklm-combined/
  src/
    background/        # MV3 service worker
    content/           # Content script bridge & styles
    inject/            # Features and helpers that run inside NotebookLM
    popup/             # Settings popup (Preact)
    shared/            # Shared utilities for settings, messaging, and keyboard
    manifest/          # manifest.config.ts (generated by CRXJS)
  _locales/            # i18n messages
  public/icons/        # Icons
  config/              # ESLint / Prettier / TS settings
  docs/                # Documentation (e.g., architecture)
  scripts/             # Release helper scripts
  vite.config.ts
  package.json
```

`dist/` is generated during build.
